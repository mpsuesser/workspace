## Commands

```text
wxt [root]            start dev server
wxt build [root]      build for production
wxt zip [root]        build for production and zip output
wxt prepare [root]    prepare TypeScript project
wxt clean [root]      clean generated files and caches
wxt init [directory]  initialize a new project
wxt submit            alias for publish-browser-extension
wxt submit init       initialize submit config/secrets
```

## Common global/dev options

```text
-c, --config <file>                   use specified config file
-m, --mode <mode>                     set env mode
-b, --browser <browser>               specify browser target
--host <host>                         dev server host
-p, --port <port>                     dev server port
-e, --filter-entrypoint <entrypoint>  only build specific entrypoints
--mv3                                 target manifest v3
--mv2                                 target manifest v2
--debug                               enable debug mode
--level <level>                       log level
-h, --help                            help
-v, --version                         version
```

## Build options

```text
wxt build [root]
  --analyze       visualize extension bundle
  --analyze-open  automatically open stats.html in browser
```

Also accepts config, mode, browser, filter-entrypoint, mv2/mv3, debug, and log level flags.

## Zip options

```text
wxt zip [root]
  --sources       always create sources zip
```

Also accepts config, mode, browser, mv2/mv3, debug, and log level flags.

## Init options

```text
wxt init [directory]
  -t, --template <template>     template to use
  --pm <packageManager>         package manager to use
```

## Prepare and clean

```sh
wxt prepare  # generates .wxt TS/env/import support
wxt clean    # removes generated files/caches
```

Use `wxt prepare --debug` to inspect hook execution order and entrypoint loader preprocessing.

## Package scripts to recommend

```json
{
  "scripts": {
    "dev": "wxt",
    "dev:firefox": "wxt -b firefox",
    "build": "wxt build",
    "build:firefox": "wxt build -b firefox",
    "zip": "wxt zip",
    "zip:firefox": "wxt zip -b firefox",
    "postinstall": "wxt prepare"
  }
}
```

## Debug flags

- Add `--debug` to WXT commands for verbose internals.
- Use `--level debug|trace|verbose` if log level control is needed.
- For module-level debug logs using `obug`, set `DEBUG=<namespace>`.
- For runner: `DEBUG=@wxt-dev/runner`.
