# alchemy@2.0.0-beta.40 (2026-05-15)

## `Cloudflare.Vite` — real Vite dev server

`bun alchemy dev` for a `Cloudflare.Vite` resource now boots a Vite dev server (`vite.createServer` via `@distilled.cloud/cloudflare-vite-plugin`) that serves client + SSR + Worker in one process — with HMR.

```ts
const web = yield* Cloudflare.Vite("Web", {
  main: "./src/worker.ts",
  env: { VITE_API_URL: api.url.as<string>() },
});
```

- HMR works in dev — edit a route component, browser hot-reloads; edit the Worker, Worker reloads.
- `props.env` keeps beta.39 semantics: `VITE_`-prefixed keys inlined into the client/SSR bundle via `import.meta.env.*`.
- Vite resolved from project `node_modules` first via `createRequire`.

## AiGateway: no more update-every-deploy

`Cloudflare.AiGateway` was reporting `update` on every deploy. Two causes:
1. Provider was diffing against `news` instead of observed cloud state.
2. Defaults didn't match Cloudflare API response shape.

Both fixed. Deploys are quiet again unless something actually changed.

## Worker HTTP: generic 500s, full Cause server-side

The Cloudflare Worker HTTP adapter was leaking internal error detail into 500 responses. Full `Cause` (stack, annotations, defects) is now logged on the server; the client receives a generic `Internal Server Error` 500. Matches Effect's own `HttpServer.serve` defaults.

## Sanitize entrypoint URLs in generated code

Codegen for Lambda, ECS, EC2-hosted, Cloudflare Worker, and Container entrypoints was interpolating `importPath` into a template string raw. Now uses `JSON.stringify(importPath)` for guaranteed-valid JS string literals (fixes Windows paths with backslashes, etc.).

## Other changes

- `Cloudflare.start` now forwards startup options to the container runtime
- `bun alchemy dev` lockfile moved to `@alchemy.run/node-utils`
- Cloudflare state store bumped to version 5 (older clients prompt to upgrade)
- `scopeTransferToStream` reverted (was causing 415 responses under workerd)
- `Axiom.Chart` gains `smartfilter` subtype
