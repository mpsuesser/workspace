# alchemy@2.0.0-beta.35 (2026-05-08)

## Headline features

### `Cloudflare.messages(queue).subscribe(...)`

Effect-native, `Stream`-shaped consumer for Cloudflare Queues. Each batch arrives as a `Stream<QueueMessage<Body>>`; success ack()s every message, failure retry()s.

```ts
yield* Cloudflare.messages<QueueMessageBody>(queueResource, {
  batchSize: 25,
  maxRetries: 3,
}).subscribe((stream) =>
  Stream.runForEach(stream, (msg) =>
    bucket.put(`/queue/${msg.body.id}`, JSON.stringify(msg.body))
      .pipe(Effect.asVoid),
  ),
);
```

Requires `Cloudflare.QueueEventSourceLive` in the provided layer.

### R2 bucket custom domains

```ts
const Assets = yield* Cloudflare.R2Bucket("Assets", {
  customDomains: ["assets.example.com", "cdn.example.com"],
});
```

### Non-string env bindings on Workers

Worker `bindings` now accept numbers, booleans, and objects directly — no `JSON.stringify`/`parse` round-trip.

```ts
const Api = yield* Cloudflare.Worker("Api", {
  main: "./src/worker.ts",
  bindings: {
    MAX_ITEMS: 100,
    DEBUG: true,
    FEATURES: { beta: true, alpha: false },
  },
});
```

`env.MAX_ITEMS` is typed as `number`, etc.

### Neon logical replication

```ts
const Project = yield* Neon.Project("App", {
  region: "aws-us-east-1",
  enableLogicalReplication: true,
});
```

In-place update on existing projects.

### CLI polish

- Version-on-npm warning at startup; skips in CI/non-TTY.
- Plain-text renderer when not in a TTY (cleaner CI logs).
- Cloud-scoped subcommands grouped under provider namespaces: `alchemy cloudflare state`, `alchemy aws state`.

## Notable fixes

- Auth providers moved into layers — stack `R` requirement is `never` again.
- `bun alchemy dev` Node compatibility and profile handling fixes.
- `QueueConsumer.listConsumers` now paginates.
- R2 custom domain interface simplified to a plain string array.
- Drizzle schema only marked updated when migrations actually drift.
- Vite `builder.sharedConfigBuild` disabled (build failure fix).
