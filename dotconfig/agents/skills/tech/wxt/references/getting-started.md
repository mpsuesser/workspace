## Bootstrap a project

```sh
pnpm dlx wxt@latest init
bunx wxt@latest init
npx wxt@latest init
```

Starter templates include Vanilla, Vue, React, Svelte, and Solid. Templates use TypeScript by default; JavaScript is possible by changing file extensions.

## From scratch minimum

1. Install WXT as a dev dependency.
2. Add at least one entrypoint, for example `entrypoints/background.ts`:

   ```ts
   export default defineBackground(() => {
     console.log('Hello world!');
   });
   ```

3. Add scripts:

   ```json
   {
     "scripts": {
       "dev": "wxt",
       "dev:firefox": "wxt -b firefox",
       "build": "wxt build",
       "build:firefox": "wxt build -b firefox",
       "zip": "wxt zip",
       "zip:firefox": "wxt zip -b firefox",
       "postinstall": "wxt prepare"
     }
   }
   ```

4. Run `wxt prepare` so TypeScript/editor support is generated, then run dev/build.

## Default project structure

```text
{rootDir}/
  .output/              # generated build artifacts
  .wxt/                 # generated WXT/TypeScript support
  assets/               # bundled assets imported by code/CSS/HTML
  components/           # auto-imported UI components
  composables/          # auto-imported Vue composables
  entrypoints/          # files WXT bundles into the extension
  hooks/                # auto-imported React/Solid hooks
  modules/              # local WXT modules, auto-discovered
  public/               # copied as-is to output
  utils/                # auto-imported project utilities
  .env                  # runtime env vars (WXT_ or VITE_ prefix)
  .env.publish          # publishing env vars
  app.config.ts         # runtime config
  package.json
  tsconfig.json
  web-ext.config.ts     # local browser startup config
  wxt.config.ts         # main WXT config
```

## Optional `src/` directory

If the project prefers source under `src`, configure:

```ts
export default defineConfig({
  srcDir: 'src',
});
```

Then source directories (`assets`, `components`, `entrypoints`, `hooks`, `utils`, `app.config.ts`) live under `src/`, while `modules/`, `public/`, `package.json`, `wxt.config.ts`, and `.env` remain at root unless otherwise configured.

## Directory customization

```ts
export default defineConfig({
  srcDir: 'src',             // default: "."
  modulesDir: 'wxt-modules', // default: "modules"
  outDir: 'dist',            // default: ".output"
  publicDir: 'static',       // default: "public"
  entrypointsDir: 'entries', // relative to srcDir, default: "entrypoints"
});
```

## Generated files and TypeScript

- `.wxt/tsconfig.json` is generated by `wxt prepare`.
- A minimal root `tsconfig.json` should usually be:

  ```jsonc
  {
    "extends": ".wxt/tsconfig.json"
  }
  ```

- If a monorepo cannot extend WXT's config, include `/// <reference path="./.wxt/wxt.d.ts" />` in the TS project.
- Do not add aliases only to `tsconfig.json`; configure `alias` in `wxt.config.ts` so both TypeScript and Vite know about them.

## Built-in aliases

| Alias | Points to | Example |
| --- | --- | --- |
| `~~` / `@@` | `<rootDir>/*` | `import "~~/scripts"` |
| `~` / `@` | `<srcDir>/*` | `import { x } from "~/utils/x"` |

## First inspection checklist

When joining an existing WXT project, inspect:

- `package.json` scripts and package manager lockfile.
- `wxt.config.ts` for `srcDir`, `modules`, `manifest`, `vite`, aliases, hooks, target config.
- `entrypoints/` names and structure.
- `modules/` for local WXT modules and hook behavior.
- `.wxt/types/imports-module.d.ts` for actual auto-imported names/import paths (run `wxt prepare` if absent).
- `web-ext.config.ts` for local dev browser behavior.
