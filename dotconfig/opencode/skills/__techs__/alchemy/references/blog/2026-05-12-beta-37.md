# alchemy@2.0.0-beta.37 (2026-05-12)

Big release. Cross-stack/stage references, typed Worker-to-Worker bindings, typed Workflows, `Alchemy.Secret`/`Variable`, cron triggers, Analytics Engine.

## Cross-stack and cross-stage references

Lazy, typed references to resources deployed by a different stack or stage. Main use case: ephemeral PR stages sharing a database with `staging`.

```ts
const project = stage.startsWith("pr-")
  ? yield* Neon.Project.ref("app-db", { stage: "staging" })
  : yield* Neon.Project("app-db", { region: "aws-us-east-1" });
```

`Resource.ref` reads attributes from the target stage's state file at plan time. Fails with `InvalidReferenceError` if the target isn't deployed.

Whole-stack reference via typed `Alchemy.Stack` handle:

```ts
export class Backend extends Alchemy.Stack<
  Backend,
  { url: string }
>()("Backend") {}

// In the consumer stack:
const backend = yield* Backend;            // same stage
const pinned = yield* Backend.stage.prod;  // pin to a specific stage
```

See [Shared database across stages](../guides/shared-database.md) and [Monorepos](../guides/monorepo.md).

## Worker-to-Worker bindings, fully typed

A Worker is bindable as a binding on another Worker; the caller gets full RPC types — no codegen, no `as` casts.

Backend Worker:

```ts
export default class Backend extends Cloudflare.Worker<Backend>()(
  "Backend",
  { main: import.meta.path },
  Effect.gen(function* () {
    const bucket = yield* Cloudflare.R2Bucket.bind(Bucket);
    return {
      hello: Effect.fn("Backend.hello")(function* (key: string) {
        const object = yield* bucket.get(key);
        return object === null ? null : yield* object.text();
      }),
      fetch: /* ... */,
    };
  }).pipe(Effect.provide(Cloudflare.R2BucketBindingLive)),
) {}
```

Bind it:

```ts
export const Website = Cloudflare.Vite("Website", {
  bindings: {
    BUCKET: Bucket,
    BACKEND: Backend,
  },
});
```

Three call shapes from the consumer:

```ts
// Option 1 — async R2 binding
const object = await env.BUCKET.get(key);

// Option 2 — Worker-to-Worker fetch
const res = await env.BACKEND.fetch(`https://backend/?key=${encodeURIComponent(key)}`);

// Option 3 — typed RPC
const backend = Cloudflare.toPromiseApi<Backend>(env.BACKEND);
const value = await backend.hello(key);
//    ^? string | null
```

Effect-native callers can `yield* Backend.bind(env.BACKEND)` for the same RPC surface without the Promise envelope.

## Workflows with typed input and output

`Cloudflare.Workflow` is now generic over input and output. Body is an `Effect.fn` taking the typed input directly.

```ts
return Effect.fn(function* (input: { roomId: string; message: string }) {
  // ...
  const stored = yield* Cloudflare.task("kv-roundtrip", /* ... */);
  yield* Cloudflare.sleep("cooldown", "2 seconds");
  return processed;
});
```

`workflow.create(input)` is type-checked end-to-end; `instance.status().output` is typed.

## Cron triggers

```ts
yield* Cloudflare.cron("0 * * * *").subscribe((controller) =>
  Effect.gen(function* () {
    yield* kv.put(`tick:${controller.scheduledTime}`, "ok");
  }),
);
```

Multiple `cron()` calls register multiple schedules on the same Worker. Granularity is 1 minute.

## Analytics Engine binding

```ts
export const Events = Cloudflare.AnalyticsEngineDataset("Events", {
  dataset: "app-events",
});

// In Worker
const analytics = yield* Cloudflare.AnalyticsEngineDataset.bind(Events);
yield* analytics.writeDataPoint({
  indexes: ["account-1"],
  blobs: ["signup"],
  doubles: [1],
});
```

Provide `Cloudflare.AnalyticsEngineDatasetBindingLive`.

## R2 buckets empty themselves on destroy

`destroy` on `R2Bucket` drains the contents before deleting. Opt out with `emptyOnDestroy: false`:

```ts
const Photos = Cloudflare.R2Bucket("Photos", { emptyOnDestroy: false });
```
