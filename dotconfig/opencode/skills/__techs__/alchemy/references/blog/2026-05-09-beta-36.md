# alchemy@2.0.0-beta.36 (2026-05-09)

Mostly Cloudflare-themed: three new Worker bindings.

## Cloudflare Images binding

Effect-native client over Cloudflare's image transformation runtime. Takes a `Stream<Uint8Array>` input and either streams a transformed image back or returns typed metadata.

```ts
export const Pipeline = yield* Cloudflare.Images({ name: "PIPELINE" });

// In Worker
const images = yield* Cloudflare.Images.bind(Pipeline);

// info()
const info = yield* images.info(request.stream);

// transform()
const transformed = yield* images
  .input(request.stream)
  .transform({ width: 512, height: 512 })
  .output({ format: "image/webp" });
return HttpServerResponse.stream(transformed.body);
```

Provide `Cloudflare.ImagesBindingLive`.

## Hyperdrive as a Worker binding

`Hyperdrive` resources are bindable on Workers like R2/KV/D1.

Over an existing origin:

```ts
const Hyperdrive = yield* Cloudflare.Hyperdrive("DbPool", {
  origin: { connectionString: process.env.POSTGRES_URL! },
});
```

Over a Neon branch:

```ts
yield* Cloudflare.Hyperdrive("app-hyperdrive", {
  origin: branch.origin,
});
```

Bind on a Worker:

```ts
const hd = yield* Cloudflare.Hyperdrive.bind(Hyperdrive);
const connectionString = yield* hd.connectionString;
```

Provide `Cloudflare.HyperdriveConnectionLive`. Requires `compatibility: { flags: ["nodejs_compat"] }` to use a `pg` client.

## R2 object lifecycle rules

```ts
const Logs = yield* Cloudflare.R2Bucket("Logs", {
  lifecycleRules: [
    {
      id: "archive-then-delete",
      prefix: "logs/",
      storageClassTransitions: [
        {
          condition: { type: "Age", maxAge: 60 * 60 * 24 * 60 },
          storageClass: "InfrequentAccess",
        },
      ],
      deleteObjectsTransition: {
        condition: { type: "Age", maxAge: 60 * 60 * 24 * 365 },
      },
      abortMultipartUploadsTransition: {
        condition: { type: "Age", maxAge: 60 * 60 * 24 * 7 },
      },
    },
  ],
});
```

Max 1000 rules per bucket. Empty array (or omitted) clears all rules.
