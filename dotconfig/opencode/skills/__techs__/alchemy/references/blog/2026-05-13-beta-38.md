# alchemy@2.0.0-beta.38 (2026-05-13)

## Cloudflare Email Routing + `send_email` binding

Four resources cover the whole flow:

```ts
// Enable Email Routing on a zone
const routing = yield* Cloudflare.EmailRouting("Routing", {
  zone: "example.com",
});

// Register a verified destination
const ops = yield* Cloudflare.EmailAddress("Ops", {
  email: "ops@example.com",
});

// Forward inbound info@ → ops@
const rule = yield* Cloudflare.EmailRule("InfoForward", {
  zone: "example.com",
  matchers: [{ type: "literal", field: "to", value: "info@example.com" }],
  actions: [{ type: "forward", value: ["ops@example.com"] }],
});

// Declare a send_email binding
export const Email = Cloudflare.SendEmail("EmailOps", {
  destinationAddress: "ops@example.com",
  allowedSenderAddresses: ["noreply@example.com"],
});
```

Bind on a Worker:

```ts
const email = yield* Cloudflare.SendEmail.bind(Email);
yield* email.send({
  from: "noreply@example.com",
  to: "ops@example.com",
  subject: "Hello",
  text: "Hi from the edge",
});
```

Methods: `send` (parsed) and `sendRaw` (RFC 822 bytes). Provide `Cloudflare.SendEmailBindingLive`.

## `Alchemy.Action` — Effects that run during apply

A new DAG node for arbitrary Effects that run as part of apply when their resolved input changes. Use for DB migrations, cache invalidation, deploy announcements, etc.

```ts
const Sync = Alchemy.Action("Sync",
  Effect.fn(function* (input: { table: string }) {
    return { rows: 42 };
  }),
);

// In a stack:
const rows = yield* Sync({ table: bucket.name });
//          ^? Output<{ rows: number }>
```

No create/update/replace/delete lifecycle — the engine hashes the resolved input and runs the body when it drifts. Removing the Action drops its persisted state in the GC pass; the body is never re-invoked on delete.

Init-style construction for bodies needing Effect Services:

```ts
const Sync = Alchemy.Action("Sync",
  Effect.gen(function* () {
    const db = yield* Database;
    return Effect.fn(function* (input: { table: string }) {
      return { rows: yield* db.count(input.table) };
    });
  }),
);
```

Plan output uses `λ` (cyan) for "will run this apply" and `·` (gray) for "input matches, skip". `--force` flips skip→run.
